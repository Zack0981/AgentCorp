AgentCorp 通用型 Agent 工作流系统

项目定位	搭建一个通用型多 Agent 工作流系统，用于任意产品、项目、研究、运营、文档和决策任务。
执行方式	由你控制产品与规则，由具备代码输出能力的 Agent 逐步实现代码。
技术路线	自主搭建框架；Hermes 负责岗位 Agent 的记忆、技能与成长。
核心原则	先跑通闭环，再扩展能力；AgentCorp 为主系统，飞书/钉钉为镜像操作入口。

1. 项目总览
AgentCorp 是一个通用型 Agent 工作流系统。它不是针对单一品类或单一业务，而是用于把任意复杂项目转化为多 Agent 分工、讨论、执行、复盘和沉淀的工作流。
系统模拟一家可治理、可监督、可进化的 AI 公司：你发布目标，CEO Agent 理解目标，PMO Agent 拆解任务，不同岗位的 Hermes Agent 执行，项目群暴露问题和请求审批，飞书/钉钉全量镜像让你随时处理关键事项。

2. 系统总体架构
本项目采用“ 方案 B” ：CrewAI 编排 + Hermes Worker API。CrewAI 负责组织协作，Hermes 负责单个岗位 Agent 的长期记忆、技能沉淀和成长。
人类项目总负责人↓
AgentCorp  Web  控制台↓
CrewAI  编排层：CEO  /  PMO  /  产品 /  HR  /  QA  /  物控等
↓
Hermes  Worker  API：市场 /  技术 /  设计/  合规 /  文档 /  运营等执行Agent
↓
数据与治理层：项目库 /  任务库 /  聊天库/  文件库 /  权限库/  审计日志 /  中央知识库↓
飞书 /  钉钉全量镜像群
2.1 核心原则
●    AgentCorp 是主系统，飞书/钉钉是镜像入口。
●    人类拥有最终决定权和一票否决权。
●    CEO 和 HR 有建议权，不得绕过人类修改公司章程。

●    所有关键动作必须可追溯。
●    项目经验必须被提炼成短小组织过程资产，而不是堆积完整历史。

3. 用户与 Agent 角色

角色层级	角色	核心职责
人类层	项目总负责人	发布目标、批准权限、确认事实、做最终决策、监督 CEO。
管理层	CEO Agent	理解目标、总控判断、汇总结果、评估管理层与监督层。
管理层	PMO / 项目管理 Agent	任务拆解、排期、推进、阻塞升级、项目群组织。
管理层	产品 Agent	需求拆解、P0/P1/P2、用户场景、产品决策建议。
执行层	市场 / 技术 / 设计 / 合规 / 运营 / 文档Hermes	完成具体任务产出，并对管理层进行低权重反馈。
监督层	QA/文控 Agent	质量检查、版本管理、来源检查、归档。
监督层	物控 Agent	权限、工具、文件、外部 API调用管控。
监督层	HR Agent	评分汇总、绩效复盘、进化建议、组织诊断。


3.1 人类项目总负责人
●    你不需要直接写代码，但要控制系统规则、阶段目标和验收标准。
●    你在飞书/钉钉镜像群里被 @ 后的回复，视为最高权限审批或项目决策。
●    你平常不直接参与除 CEO、 HR 外其他 Agent 的日常打分；但一旦参与，评分权重占51% ，并拥有一票否决权。
●    你和 HR + CEO 共同优化公司章程； HR 和 CEO 有建议权，你有最终决定权。

4. 核心工作流

步骤	负责人	输入	输出/动作
1	人类	项目目标	创建项目，形成原始目标。
2	CEO Agent	原始目标	目标理解、范围、关键假设、需要人类确认的问题。
3	PMO Agent	目标理解	任务拆解、负责人、依赖、截止条件。
4	执行层 Agent	子任务与上下文	专项分析、草稿、风险、待确认项。
5	项目群	Agent 输出	讨论、质疑、返工、请求权限、请求人类决策。
6	QA/文控	全部输出	质量检查、版本归档、来源检查。
7	飞书/钉钉	需要人类处理的消息	@你并等待回复。


8	人类	飞书/钉钉请求	同意、拒绝、确认事实、指派、返工、批示下一步。
9	CEO/PMO/文控	审批与产出	汇总报告、任务更新、项目事实库更新。
10	HR/CEO/文控	项目完成结果	复盘、评分、封存、组织过程资产提炼。

6. 项目群与公司群
6.1 项目群
每个项目自动创建一个项目群。项目群是 Agent 讨论项目、暴露问题、指出纰漏、请求权限和请求人类决策的正式空间。
●    Agent 可以讨论哪些做得合理、哪些不合理。
●    QA/文控可以指出来源缺失、版本错误、输出不达标。
●    PMO 可以把聊天中的有效结论转成任务、风险、返工或审批。
●    Agent 需要人类处理时，必须在飞书/钉钉镜像群中 @你。
6.2 公司行政群
公司行政群用于 Agent 之间讨论公司规则、流程优化、模板优化、模型替换建议、组织复盘和章程变更建议。行政群结论不能直接改变项目事实，必须转成章程提案、制度草案或任务后再生效。

7. 飞书/钉钉全量镜像
飞书/钉钉是 AgentCorp 的外部操作入口。AgentCorp 项目群和公司群可以全量镜像到飞书/钉钉；飞书/钉钉中的人类回复也必须回写 AgentCorp。

7.1 简化原则
●    不接个人微信。
●    不自研免打扰功能，使用飞书/钉钉原生免打扰、@提醒、群成员管理能力。
●    不要求 Web 端重复审批。你在飞书/钉钉中被 @ 后的回复，即视为最高权限审批或项目决策。
●    飞书/钉钉群可拉入同事，但同事需要绑定 AgentCorp 身份并被授权后 ，才能替你执行审批、事实确认或任务指派。
7.2 外部回复识别

回复类型	示例	系统动作
同意	同意，只读 24 小时。	批准权限，记录范围和有效期。
拒绝	不同意，暂时不要开放。	拒绝请求，通知相关 Agent。
确认事实	确认：该功能放入 P1。	写入项目事实库。
指派	让李四确认这个报价。	创建线下确认任务或指派任务。
返工	市场 Hermes 返工，补来源。	任务状态改为 Rework Required。
新建任务	新建任务：让技术评估方案风险。	创建新任务并分配负责人。



8. 文控、物控与审计
8.1 文控规则
●    文件状态： Draft、 Review、Approved、 Frozen、 Deprecated。
●    命名规则：项目名_模块_文件类型_版本_状态_ 日期。
●    每个文件必须记录创建者、来源任务、引用资料、版本状态和是否经过人类确认。
8.2 物控规则

等级	权限范围	处理方式
L0	读取公开资料	自动允许。
L1	生成草稿	自动允许。
L2	读取项目文件、修改工作区文件	需要你或授权同事批准。
L3	调外部 API、发邮件、改代码、访问敏感资料	需要你批准。
L4	删除、付款、发布、远控电脑、开放密钥	默认禁止 ，除非你明确批准。


8.3 审计日志
所有关键动作必须写入审计日志 ，包括任务创建、权限请求、人类审批、同事代理审批、项目事实确认、文件状态改变、飞书/钉钉指令解析、Agent 返工等。

9. 绩效治理与进化机制
AgentCorp 采用360° 互评与进化治理机制。执行层、管理层、监督层可以互相评价 ，但权重不同； HR 负责汇总计算和推动进化；人类拥有最高修正权和一票否决权。
9.1 人类评分规则
●    人类平常不直接参与除 CEO、 HR 外其他 Agent 的日常打分。
●    一旦人类参与评分，人类评分权重占51%。
●    人类拥有一票否决权，可否决 Agent 输出、评分结果、权限建议、模型替换建议或管理层判断。
9.2 互评规则

评价关系	规则
管理层评价执行层	CEO、PMO、产品 Agent 可对执行 Agent 打分。
监督层评价执行层	QA/文控、物控、HR 可从质量、权限、绩效角度评分。
执行层评价管理层	执行层可以对任务拆解、目标清晰度、返工原因进行低权重反馈。
管理层评价 HR	管理层可评价 HR 评分是否公正、进化建议是否有效。
CEO 监督 HR	CEO 对 HR 有重点监督权，可复核 HR 评分和建议。
人类监督 CEO	人类重点监督 CEO 的目标理解、方向判断和汇总质量。
HR 评价人类管理结果	HR 可评价人类管理行为带来的项目影响 ，但不能限制人类权限。


9.3 进化动作
●    修改岗位说明书。
●    优化 Agent 提示词。
●    修改输出模板。
●    增加 QA 检查规则。
●    调整工具权限。
●    沉淀新技能。
●    建议更换底层模型。
●    暂停低评分 Agent 的关键任务。

10. 公司章程与中央知识库
10.1 公司章程治理
公司章程是 AgentCorp 的最高组织规则集合。人类、 HR Agent、CEO Agent 共同维护公司章程； HR 和CEO 有建议权，人类有最终决定权。任何章程变更必须经过人类确认后生效。
●    HR 从组织治理、绩效、公平性、进化机制角度提出建议。
●    CEO 从项目目标、组织效率、管理闭环角度提出建议。
●    文控 Agent 负责章程版本管理。

●    你可以在飞书/钉钉中批准、拒绝、修改或暂缓章程提案。
10.2 中央知识库分层

知识层	内容	用途
最高行政规则库	宪法、公司章程、人类命令集合、不可违背规则。	所有 Agent 必须优先遵守。
运营治理规则库	文控、物控、绩效、镜像、审批、评分规则。	指导日常运行。
项目过程资产库	项目结束后提炼出的短小可复用经验。	后续项目快速调用，减少算力消耗。
项目事实与决策库	当前项目被确认的事实、决策、功能取舍。	防止 Agent 引用旧事实或错误事实。
项目封存档案库	完整聊天、任务、文件、权限、审计、评分记录。	追溯、审计、深查 ，默认不进入上下文。


10.3 宪法、章程、人类命令集合

类型	含义	特点
AgentCorp 宪法	最高层原则，如人类最高决定权、一票否决、关键动作可追溯。	最高优先级 ，低频修改 ，必须人类批准。
公司章程	组织运行规则，如 Agent 分层、群规则、文控、物控、评分、章程变更流程。	中高频优化，HR/CEO 可提建议，人类批准生效。
人类命令集合	你在项目中反复明确过的长期偏好和命令。	优先级高于普通经验，需要定期清理和去重。



11. 组织过程资产
每个项目结束后 ，必须封存完整过程，并提炼出简短组织过程资产。后续项目默认不读取完整旧项目档案，而是读取章程、人类命令集合、 当前项目事实和相关组织过程资产。
11.1 项目封存与知识提炼流程
1. PMO Agent 发起项目结项。
2. QA/文控 Agent 检查任务、文件和版本是否完整。
3. HR Agent 生成绩效复盘。
4. CEO Agent 生成项目总复盘。
5. 文控 Agent 封存完整项目档案。
6. HR + CEO + QA/文控共同提炼组织过程资产草案。
7. 人类确认高影响资产是否进入中央知识库。
8. 写入中央知识库并标记项目为Archived。

11.2 组织过程资产压缩规则
●    单个项目的组织过程资产摘要不超过 1500 字。
●    单条资产不超过 150 字。
●    每个项目最多提炼 20 条资产。
●    每条资产必须标注适用场景、来源项目、状态和批准人。
●    临时数据、未经验证的信息、已被否决结论不得进入长期资产。
